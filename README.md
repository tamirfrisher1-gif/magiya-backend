# MAGIYA — Smart Wedding Guest Management Platform

> No spreadsheets. No manual tracking. No middlemen.  
> **Live demo → [magiya-frontend.onrender.com](https://magiya-frontend.onrender.com)**

MAGIYA is a full-stack wedding guest management platform. Couples sign up, import their contacts from Google, swipe to build their guestlist, generate an AI invitation photo, send personal Telegram RSVP links, and watch responses roll in on a live dashboard — all the way to automatic seating arrangement.

---

## How it works

1. Couple signs up → gets a unique Wedding ID
2. Imports Google Contacts via OAuth → contacts appear as swipe cards
3. Swipes right to invite, left to skip → guests sorted into groups
4. Generates an AI invitation photo (OpenAI `gpt-image-1`) in one sentence
5. Sends a personal Telegram link to each guest → bot collects RSVP automatically
6. Live dashboard updates in real-time → seating plan auto-fills from confirmed guests

---

## Features

- **Google Contacts Import** — OAuth 2.0, one click, zero manual entry
- **Swipe to Invite** — Tinder-style card UI to sort contacts into groups
- **AI Invitation Photo** — Generated by OpenAI `gpt-image-1` from a one-line description
- **Telegram RSVP Bot** — Personal links per guest, bot collects attendance + headcount + dietary needs
- **Live Dashboard** — Real-time stats from Supabase (invited / attending / declined / awaiting)
- **Seating Plan** — Drag & drop, auto-populated from confirmed guests only

## Architecture & Deployment

The platform runs as two services on Render:

| Service | URL |
|---|---|
| Backend API | https://magiya-api.onrender.com |
| Frontend | https://magiya-frontend.onrender.com |

Frontend (static HTML/JS) calls the FastAPI backend.  
Backend connects to Supabase (database + realtime), OpenAI (AI photo generation), and Telegram (RSVP bot).  
Both services auto-deploy from `main` on every push — zero manual deployment.

## Database Schema (Supabase / PostgreSQL)

```sql
-- Wedding profile
create table weddings (
  id                   text primary key,   -- ex: "sarah-et-david-15-08-2026"
  bride_name           text not null,
  groom_name           text not null,
  wedding_date         date not null,
  venue                text,
  contact_email        text,
  table_capacity       int  not null default 10,
  invitation_text      text,
  invitation_image_url text
);

-- Guest groups (Work, Family, Friends…)
create table groups (
  id          uuid primary key default gen_random_uuid(),
  wedding_id  text not null references weddings(id) on delete cascade,
  name        text not null,
  unique (wedding_id, name)
);

-- Guests
create table guests (
  id          uuid primary key default gen_random_uuid(),
  wedding_id  text references weddings(id) on delete cascade,
  full_name   text not null,
  phone       text,
  group_name  text,
  invited     boolean,
  unique (phone, wedding_id)
);

-- RSVPs
create table rsvps (
  id                   uuid primary key default gen_random_uuid(),
  guest_id             uuid references guests(id),
  status               text check (status in ('confirmed','declined','pending')) default 'pending',
  party_size           int default 1,
  dietary_restrictions text,
  responded_at         timestamptz,
  unique (guest_id)
);

-- Seating assignments
create table seating_assignments (
  id           uuid primary key default gen_random_uuid(),
  wedding_id   text not null references weddings(id) on delete cascade,
  guest_phone  text not null references guests(phone) on delete cascade,
  table_number int  not null,
  unique (wedding_id, guest_phone)
);
```

## Local Setup

### 1. Clone

```bash
git clone https://github.com/tamirfrisher1-gif/magiya-backend.git
cd magiya-backend
```

### 2. Virtual environment

```bash
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # macOS / Linux
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Environment variables

```bash
cp .env.example .env
```

```env
TELEGRAM_BOT_TOKEN=        # @BotFather on Telegram
SUPABASE_URL=              # Supabase → Project Settings → API
SUPABASE_KEY=              # Supabase → Project Settings → API
OPENAI_API_KEY=            # platform.openai.com
GOOGLE_CLIENT_ID=          # Google Cloud Console → OAuth 2.0
GOOGLE_CLIENT_SECRET=      # Google Cloud Console → OAuth 2.0
```

### 5. Run locally

```bash
uvicorn main:app --reload
```

Frontend: open `frontend/index.html` in a browser, or use the live URL above.
